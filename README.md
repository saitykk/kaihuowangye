# 🍳 厨房每日开货系统 (Kitchen Stock & Order System)

> 专为餐饮门店后厨设计的纯前端、轻量级日常开货（订货）工具。

这是一个为解决后厨日常高频采购需求而开发的极速开货网页应用。无需下载 App，无需配置复杂的服务器与数据库，仅依靠手机自带的浏览器即可实现高效的菜品管理、采购统计以及一键生成按供应商分类的采购清单。

## ✨ 核心功能与作用

### 1. ⚡️ 极速订货与一键结算
* **快速检索**：支持顶部搜索框模糊搜索，或通过横滑分类导航快速定位货品。
* **智能填入**：点击“选择”按钮自动填入预设的默认采购数量（如：默认进货 10kg）[span_0](start_span)[span_0](end_span)，支持点击高亮区域快速手动改量。
* **一键成单**：自动按“供应商”将已选货品分类汇总，一键复制排版整齐的文本清单（支持自动添加店铺抬头），可直接粘贴至微信发送给供货商。

### 2. 📦 灵活的自定义基础库
系统内置了包含牛羊骨、预制熟食、蔬菜类等在内的近 60 项核心货品基础数据[span_1](start_span)[span_1](end_span)。同时提供了完善的 **⚙️ 管理模式**，允许用户在手机端自由掌控数据：
* 自定义添加、删除、重命名 **供应商** 与 **货品类目**。
* 灵活管理货品字典，支持设置不同的计价单位（kg、件、包、箱等）[span_2](start_span)[span_2](end_span) 与起订量。
* 采用 13 位绝对时间戳作为货品唯一 ID，彻底杜绝数据覆盖冲突。

### 3. 💾 极致的本地运行与隐私保护
* **纯前端无后端架构**：所有货品信息与购物车状态均保存在当前设备浏览器的 `LocalStorage` 中。
* **零延迟**：不依赖网络请求加载数据，即使在后厨信号不佳的环境下依然拥有原生 App 般的流畅操作体验。

### 4. 🔄 智能的数据管理与同步
由于数据隔离在各自的手机设备中，系统专门设计了两套多端协作方案：
* **无感增量升级（温柔合并）**：当项目管理者在 GitHub 源码中更新了统一的核心菜单（提升代码版本号）时，员工刷新网页会自动下载新增菜品，**且完美保留员工在自己手机上额外添加的“私有备注货品”**。
* **一键导入 / 导出**：支持将整店数据一键导出为 JSON 文本，通过微信发送给新员工。新员工在网页内点击“覆盖导入”即可瞬间克隆完整的店铺货品库。

## 📱 界面交互亮点
* 采用移动端优先的设计，适配所有手机屏幕。
* 顶部操作栏与底部结算栏自动吸顶/吸底，防止长列表滚动时迷失焦点。
* 购物车状态实时视觉反馈（选中状态卡片变蓝、底部实时计算已选项数）。
* 内置防误触机制与“清空开货单”操作提示，保障每日采购流程的安全与整洁。

## 🛠️ 技术栈
* **HTML5 + CSS3 + 原生 JavaScript (ES6)**
* 无任何第三方依赖框架，轻量、稳定、易于二次开发与部署。
