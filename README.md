# 👨‍🍳 厨房每日开货系统 (Kitchen Stock System)

这是一个专为餐饮门店和后厨团队打造的**极简开货（订货）网页应用**。
纯前端运行，无需配置服务器或数据库，数据全部本地化存储，并支持跨设备的“温柔合并”云端菜单分发，解决日常高频、繁琐的后厨订货对账需求。

## ✨ 核心亮点

* 📱 **纯净移动端体验**：专为手机浏览器优化，操作丝滑无卡顿，支持侧滑导航与一键吸顶。
* ☁️ **零成本托管**：基于纯 HTML/JS/CSS 构建，可完美免费部署在 GitHub Pages 或 Vercel 等纯静态托管平台。
* 💾 **本地存储与私有化**：依托浏览器的 `LocalStorage` 进行数据存储，员工点单互不干扰，保护隐私且响应极速。
* 🤝 **“温柔合并”版本控制**：管理员在代码中更新基础菜单并升级版本号后，系统会自动将新菜品“静默下发”到所有员工手机中，且**绝对不会**覆盖或删除员工自己手动添加的私有菜品。
* 🧾 **一键生成结算单**：自动按“供应商”对选中的菜品进行分类汇总，并支持自动添加“店铺抬头”，一键复制发送微信群。
* 📦 **数据导入与导出**：一键生成 JSON 备份文本，方便在新手机上瞬间恢复或克隆整套系统数据。

## 🚀 极速部署指南 (GitHub Pages)

无需懂任何代码，只需三步即可生成你的永久免费网址：

1. 登录 GitHub，创建一个新的 Public 仓库。
2. 将本仓库中的 `index.html` 文件上传至你的仓库。
3. 点击仓库顶部的 **Settings** -> 左侧菜单找 **Pages** -> 在 Source 中选择 `Deploy from a branch` -> Branch 选择 `main` 并保存。
4. 等待 1-2 分钟，即可获得你的永久访问链接（例如：`https://你的用户名.github.io/仓库名/`）。

## 🛠️ 日常维护与菜单升级

本系统内置了基于时间戳的防撞车 ID 机制与版本比对逻辑。当需要为全店统一增加新菜品或新供应商时，请按照以下步骤操作代码：

1. 打开 GitHub 中的 `index.html`，点击 ✏️ 编辑图标。
2. 找到代码中的 `const PRESET_DATA = { ... }` 区域。
3. 按照现有格式插入新的货品信息。**重要提示：** 请务必使用 **13位纯数字（如时间戳 `1783150000999`）** 作为新货品的 `id`，严禁使用短数字，以防与员工本地自建的货品 ID 冲突。
4. 找到紧接着下方的版本号变量：`const CODE_VERSION = 1.1;`，将其改大（例如改为 `1.2`）。
5. 滚动到页面底部点击 **Commit changes** 保存。员工刷新网页后即可无感接收新菜单。

## ⚠️ 极其重要的注意事项

1. **绝对不要清理浏览器缓存！** 本网页的所有数据（包括添加的菜品、选好的购物车）均保存在手机当前浏览器的本地缓存中。一旦清理缓存或卸载重装浏览器，数据将永久丢失。
2. **建议使用固定浏览器：** 强烈建议使用手机系统自带的 Safari 或默认浏览器将其“添加到主屏幕”。如果在微信内打开，退出微信时极易被系统自动清空缓存。
3. **日常备份：** 建议管理员定期在管理面板中使用“导出备份”功能，将代码发送至微信“文件传输助手”留存。

## 👨‍💻 技术栈

* HTML5 + CSS3 (原生实现，无 UI 框架依赖)
* Vanilla JavaScript (纯原生 JS，无 Vue/React 依赖)
* LocalStorage API
